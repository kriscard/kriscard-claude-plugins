# Blog Post Templates

## Tutorial Post (1200-2500 words)

```markdown
# [How to / Building] [Topic] with [Technology]

[Personal hook - why you're writing this]

## Prerequisites
- [Prerequisite 1]
- [Basic knowledge of X]

## [Step 1: Setup]
[Explanation]
[Code block]
[What this accomplishes]

## [Step 2: Core Implementation]
[Continue pattern...]

## Common Pitfalls
- **[Pitfall 1]**: [How to avoid]

## What We Built
[Summary and final code]

## Next Steps
- [Related topic to explore]
```

## Project Showcase (800-1500 words)

```markdown
# [Project Name]: [Key Feature]

[Hook - the problem this solves]

## The Problem
[What pain point drove this project]

## Tech Stack
| Technology | Why I Chose It |
|------------|----------------|
| [Tech 1] | [Rationale] |

## Key Implementation Details
### [Challenge 1]
[How you solved it]

## What I Learned
- [Lesson 1]

## Try It Out
- [Live demo]
- [GitHub repo]
```

## Opinion Post (600-1200 words)

```markdown
# [Clear Thesis Statement]

[Bold opening stating your position]

## [The Conventional Wisdom]
[Acknowledge common perspective]

## [Your Counter-Argument]
[Main point with evidence]

## [Acknowledging Trade-offs]
[Where the other side is valid]

## The Bottom Line
[Clear takeaway]
```

## TIL Post (300-500 words)

```markdown
# TIL: [Specific Thing]

[One-line context]

## The Problem
[What you were trying to do]

## The Solution
[Code snippet]

## Why This Works
[Brief explanation]
```

## Comparison Post (1000-1800 words)

```markdown
# [A] vs [B]: [Comparison Point]

[Context and quick verdict]

## Quick Comparison
| Aspect | [A] | [B] |
|--------|-----|-----|

## When to Choose [A]
- [Scenario 1]

## When to Choose [B]
- [Scenario 1]

## My Recommendation
[Your verdict]
```

## Selection Guide

| You have... | Use |
|-------------|-----|
| Step-by-step instructions | Tutorial |
| Something you built | Project Showcase |
| Strong opinion | Opinion |
| Quick tip | TIL |
| Two options to compare | Comparison |
